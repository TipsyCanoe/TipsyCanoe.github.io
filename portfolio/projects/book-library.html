Book Library Classifier

AI-powered personal library mannagament system

GitHub

A full-stack application that catalogs personal book collections with automatic genre classification. Built to solve the problem of organizing a mixed library of novels, manga, textbooks, and more.

The system uses Groq's Llama 3.3 70B model combined with Google Books API metadata to accurately classify books into predefined genres, then stores everything in a PostgreSQL database with a React frontend for browsing and management.

Features

  • AI Classification — Automatic genre detection using LLM with Google Books descriptions for context
  • Batch Processing — CSV import for bulk library cataloging with rate limiting
  • Series Caching — Books in the same series get consistent classification
  • Read/Unread Tracking — Track reading progress across your collection
  • Genre Filtering — Filter and sort by genre, author, read status
  • Manual Override — Edit genres and metadata via the UI when AI gets it wrong

Challenges

Inconsistent LLM Output

The LLM would sometimes return verbose explanations instead of structured data. Solved by tightening the system prompt, adding examples, and implementing robust parsing that extracts data regardless of format variations.

Series Consistency

Books in the same series (e.g., Dungeon Crawler Carl) were getting classified into different genres. Implemented a series cache that applies the first book's classification to subsequent entries.

Manga Classification

Manga was being classified by story content (Fantasy, Action) rather than format. Added a format override in the CSV that bypasses the LLM for known formats.

What I learned

  • LLM prompt engineering for structured output
  • Rate limiting and batch processing strategies
  • Integrating multiple external APIs in a single pipeline
  • PostgreSQL schema design and migrations
  • Full-stack React + Flask development
